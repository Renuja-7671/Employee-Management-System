# Fix: Employee Names Display as Null in Admin Leave Requests Page

## Summary

Fixed employee names showing as null in the admin portal's Leave Requests page by updating the employee name transformation logic to use proper field fallback order.

## Problem

Newly added employees' names were displaying as "null" or empty in the admin Leave Requests page because the component was only checking deprecated `firstName` and `lastName` fields. Newly added employees use `fullName` and `callingName` fields instead.

## Root Cause

In `/src/components/admin/LeaveManagement.tsx`, employees were transformed with:
```typescript
name: `${emp.firstName} ${emp.lastName}`
```

For newly added employees where `firstName` and `lastName` are null, this resulted in "null null" display.

## Solution

Updated the transformation logic to check multiple name fields with proper fallback:
```typescript
name: emp.fullName 
      || emp.callingName 
      || `${emp.firstName || ''} ${emp.lastName || ''}`.trim() 
      || emp.employeeId
```

## Changes Made

**File**: `/src/components/admin/LeaveManagement.tsx`
- **Lines**: 127-130
- **Change**: Updated employee name transformation to use proper fallback logic

## Impact

- ✅ Newly added employees now display with proper names
- ✅ Legacy employees continue to work as before
- ✅ Names never show as null (falls back to employeeId)
- ✅ Affects all 7 locations using employee names:

  - Export CSV/PDF functions
  - Approval/Decline dialogs
  - Cover employee display
  - Covering duties display
